Perceived Performance

Eli Fitch

objective time

vs subjective time

  • we tend to optimize for objective time and it's easier to test/measure
  • what if users don't notice?
    • there has to be enough of a difference for people to tell the difference
    • has to be more than 20% improvement for users to notice and 30% for it to make a real difference
  • focus on subjective for better bang for the buck

active vs passive

  • takes about 1 sec for user to transition
  • either keep users active or make the passive portion feel shorter

Keep users active

  • hover animations
  • optimistically update the UI and make API calls in the background
  • optimistically fire events when user clicks down or approaches button
  • if user is in a funnel, preload the next page while user is interacting with current page

loading spinners

and progress bars

  • it's a clue that users are waiting and should go into passive state
  • use if wait time will be 1-4 seconds
  • progress bars
    • better because it gives users an idea of how long is left
    • keep it "real-ish" by measuring user response time and comparing it to expected time and using the ratio to adjust other expected load times

longer waits

use multiple loading screens that enagage the user to make them feel like they're doing something
